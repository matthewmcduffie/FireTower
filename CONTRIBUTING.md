# Contributing to FireTower

Contributions are welcome. If you have found a bug, have an idea for an improvement, or want to add support for a new virtualization provider, this is the place to start.

---

## How to Contribute

**For bug reports and feature requests**, please open an [Issue](../../issues). Include as much detail as you can — what you expected to happen, what actually happened, and the version of FireTower you are running.

**For code changes**, please open a [Pull Request](../../pulls). Fork the repository, make your changes on a dedicated branch, and submit a PR with a clear description of what the change does and why. Smaller, focused pull requests are easier to review and more likely to be merged quickly.

---

## A Few Things to Keep in Mind

- FireTower is a native Windows desktop application. Please do not convert it toward web technologies or cross-platform frameworks.
- All new code should build without warnings and, where practical, include tests.
- The original author is Matthew McDuffie. The MIT license requires that copyright attribution be preserved in any copies or derivative works.

---

Thank you for taking the time to improve FireTower.
